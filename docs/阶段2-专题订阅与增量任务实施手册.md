## 0. 先认识后台任务

浏览器请求应尽快返回；下载论文这类耗时操作则交给**后台任务**。任务队列像待办清单：API 创建一张任务单，worker 取走并执行，数据库保存任务状态。**幂等键**则让“同一件事重复点击”仍只产生一份有效任务。

下面的例子先不用 Celery，只展示状态为何要单独保存：

```python
"""演示一个最小任务状态机。"""

job = {"status": "pending"}


def start_job(item: dict[str, str]) -> None:
    """将待执行任务标记为运行中。

    :param item: 保存任务状态的可变字典。
    :raises ValueError: 任务不处于待执行状态时抛出。
    """
    if item["status"] != "pending":
        raise ValueError("任务不能重复启动")
    item["status"] = "running"
```

调用后 `job["status"]` 变为 `"running"`。本章再将这个字典替换为 `CollectionJob` 表，并由 Celery worker 完成真实采集。

## 0. 范围、技术选择与开始条件

### 固定技术选择

| 领域 | 选择 | 原因 |
| --- | --- | --- |
| Broker / Result Backend | Redis | 部署简单，适合任务消息、延迟任务和短期状态。 |
| Task Queue | Celery | Python 后端常见，具备 worker、retry、beat 调度与可观测性基础。 |
| 调度 | Celery Beat | 订阅更新频率应是任务调度，而不是在 Web 请求内轮询。 |
| 持久化真相来源 | PostgreSQL | Job 状态、统计和错误摘要必须落库，不能只依赖 Redis 结果过期时间。 |

在开始前确认阶段 1 已完成：专题有稳定 ID，`/api/v1/topics` 可用，Alembic 迁移已接管表结构。

### 本阶段不做什么

- 不做 PDF 全文下载和解析；只处理元数据与摘要增量。
- 不做分布式多地区调度、优先级队列或复杂工作流编排。
- 不在任务中直接生成 RAG 回答。
- 不把 Celery/Redis 当作唯一数据源；最终状态必须写回 PostgreSQL。

### 1.2 状态机

统一使用：

```text
pending → running → succeeded
                  ↘ failed
pending/running → cancelled（仅在真正实现取消时开放）
```

规则：

- `succeeded`、`failed`、`cancelled` 是终态，不能重新改回 `running`。
- 重试应创建新的 attempt 记录，或明确记录 `retry_count` 与每次错误；不要覆盖原失败原因。
- 同一专题、同一订阅、同一采集窗口只能存在一个活跃任务。
- `idempotency_key` 由 API 或应用服务生成；重复提交必须返回已有活跃 Job，而不是重复采集。

本节在阶段 1 的专题和 `/api/v1` 基础上继续。代码的重点是：API 只创建 Job，worker 执行采集，PostgreSQL 保存最终状态。

```python
from datetime import datetime
from enum import StrEnum
from uuid import UUID, uuid4

from pydantic import BaseModel, Field

class JobStatus(StrEnum):
    """定义采集任务的可用状态。"""

    PENDING = "pending"
    RUNNING = "running"
    RETRY_PENDING = "retry_pending"
    SUCCEEDED = "succeeded"
    FAILED = "failed"
    CANCELLED = "cancelled"


TERMINAL_STATUSES = {JobStatus.SUCCEEDED, JobStatus.FAILED, JobStatus.CANCELLED}


class CollectionJob(BaseModel):
    """表示一次可追溯的论文采集任务。"""

    id: UUID = Field(default_factory=uuid4)
    topic_id: UUID
    subscription_id: UUID | None = None
    status: JobStatus = JobStatus.PENDING
    trigger_type: str = "manual"
    idempotency_key: str
    fetched_count: int = 0
    created_count: int = 0
    updated_count: int = 0
    failed_count: int = 0
    retry_count: int = 0
    error_message: str | None = None
    created_at: datetime | None = None
    started_at: datetime | None = None
    finished_at: datetime | None = None

    def can_transition_to(self, target: JobStatus) -> bool:
        return (self.status, target) in {
            (JobStatus.PENDING, JobStatus.RUNNING),
            (JobStatus.RETRY_PENDING, JobStatus.RUNNING),
            (JobStatus.RUNNING, JobStatus.SUCCEEDED),
            (JobStatus.RUNNING, JobStatus.FAILED),
            (JobStatus.RUNNING, JobStatus.RETRY_PENDING),
            (JobStatus.PENDING, JobStatus.CANCELLED),
            (JobStatus.RETRY_PENDING, JobStatus.CANCELLED),
        }
```

`StrEnum` 让状态既是枚举成员又可作为字符串保存；`Field(default_factory=uuid4)` 会在未传入 `id` 时生成 UUID。`trigger_type` 记录任务来源：`manual` 为用户触发、`scheduled` 为定时触发、`retry` 为重试触发。

#### 应用服务：创建 Job 而不是直接采集

创建 `app/application/collection_job_service.py`：

```python
from datetime import UTC, datetime
from uuid import UUID

from app.domain.jobs.models import CollectionJob
from app.domain.jobs.ports import CollectionJobRepository, TaskDispatcher


class CollectionJobService:
    """创建任务并通过调度器提交执行。"""

    def __init__(self, repository: CollectionJobRepository, dispatcher: TaskDispatcher) -> None:
        self.repository = repository
        self.dispatcher = dispatcher

    def create(
        self,
        topic_id: UUID,
        *,
        idempotency_key: str,
        trigger_type: str = "manual",
        subscription_id: UUID | None = None,
    ) -> tuple[CollectionJob, bool]:
        candidate = CollectionJob(
            topic_id=topic_id,
            idempotency_key=idempotency_key,
            trigger_type=trigger_type,
            subscription_id=subscription_id,
            created_at=datetime.now(UTC),
        )
        job, created = self.repository.create_if_absent(candidate)
        if created:
            try:
                self.dispatcher.dispatch_collection(job.id)
            except Exception as exc:
                self.repository.finish_failure(job.id, f"unable to enqueue task: {exc}")
                raise
        return job, created

    def get_or_raise(self, job_id: UUID) -> CollectionJob:
        job = self.repository.get(job_id)
        if job is None:
            raise LookupError("collection job not found")
        return job
```

```python
from uuid import UUID, uuid4

from app.application.collection_job_service import CollectionJobService
from app.domain.jobs.models import CollectionJob, JobStatus

TOPIC_ID = uuid4()


class FakeCollectionJobRepository:
    def __init__(self) -> None:
        self.by_key: dict[str, CollectionJob] = {}

    def create_if_absent(self, job: CollectionJob) -> tuple[CollectionJob, bool]:
        existing = self.by_key.get(job.idempotency_key)
        if existing is not None:
            return existing, False
        self.by_key[job.idempotency_key] = job
        return job, True

    def get(self, job_id: UUID) -> CollectionJob | None:
        return next((job for job in self.by_key.values() if job.id == job_id), None)

    def finish_failure(self, job_id: UUID, message: str) -> CollectionJob:
        job = self.get(job_id)
        assert job is not None
        job.status = JobStatus.FAILED
        job.error_message = message
        return job


class FakeDispatcher:
    def __init__(self) -> None:
        self.calls: list[UUID] = []

    def dispatch_collection(self, job_id: UUID) -> None:
        self.calls.append(job_id)


def test_duplicate_key_reuses_existing_job() -> None:
    repository = FakeCollectionJobRepository()
    dispatcher = FakeDispatcher()
    service = CollectionJobService(repository, dispatcher)

    first, first_created = service.create(TOPIC_ID, idempotency_key="topic:window-1")
    second, second_created = service.create(TOPIC_ID, idempotency_key="topic:window-1")

    assert first.id == second.id
    assert first_created is True
    assert second_created is False
    assert dispatcher.calls == [first.id]
```

### 2.2 PostgreSQL 与 Alembic

创建 Subscription、CollectionJob、IndexJob 的 ORM 和迁移。最低数据库约束：

- `subscriptions.topic_id` 外键指向专题。

- `status`、`created_at`、`started_at`、`finished_at` 建组合查询索引。

- 统计字段至少包含 `fetched_count`、`created_count`、`updated_count`、`failed_count`。

Repository 的关键是原子状态切换。以 `mark_running` 为例：

```python
from datetime import UTC, datetime
from sqlalchemy import update

def mark_running(self, job_id):
    with self.session_factory() as session:
        statement = (
            update(CollectionJobRecord)
            .where(CollectionJobRecord.id == job_id)
            .where(CollectionJobRecord.status.in_(("pending", "retry_pending")))
            .values(status="running", started_at=datetime.now(UTC))
            .returning(CollectionJobRecord)
        )
        record = session.execute(statement).scalar_one_or_none()
        session.commit()
        return None if record is None else to_domain(record)
```

`schedule_retry()` 使用同样的条件更新：只允许 `running -> retry_pending`，同时将 `retry_count` 加一并写入 `error_message`；`finish_failure()` 只允许 `running -> failed`。这两个约束要与上面的状态机测试一起覆盖，避免 worker 重复执行时覆盖终态。

只有成功从 `pending` 或 `retry_pending` 更新为 `running` 的 worker 才继续采集；其他重复投递的 worker 应直接退出。

```python
from datetime import UTC, datetime
from uuid import UUID

from sqlalchemy import select, update
from sqlalchemy.exc import IntegrityError

from app.domain.jobs.models import CollectionJob, JobStatus
from app.infrastructure.persistence.job_models import CollectionJobRecord


def to_job(record: CollectionJobRecord) -> CollectionJob:
    return CollectionJob.model_validate(record, from_attributes=True)


class PostgresCollectionJobRepository:
    def __init__(self, session_factory) -> None: self.session_factory = session_factory

    def create_if_absent(self, job: CollectionJob) -> tuple[CollectionJob, bool]:
        with self.session_factory() as session:
            try:
                session.add(CollectionJobRecord(**job.model_dump())); session.commit()
            except IntegrityError:
                session.rollback()
                record = session.scalar(select(CollectionJobRecord).where(CollectionJobRecord.idempotency_key == job.idempotency_key))
                return to_job(record), False
            return job, True

    def get(self, job_id: UUID) -> CollectionJob | None:
        with self.session_factory() as session:
            record = session.get(CollectionJobRecord, job_id)
            return None if record is None else to_job(record)

    def mark_running(self, job_id: UUID) -> CollectionJob | None:
        return self._transition(job_id, (JobStatus.PENDING, JobStatus.RETRY_PENDING), JobStatus.RUNNING, started_at=datetime.now(UTC))

    def finish_success(self, job_id: UUID, *, fetched: int, created: int, updated: int) -> CollectionJob:
        job = self._transition(job_id, (JobStatus.RUNNING,), JobStatus.SUCCEEDED, fetched_count=fetched, created_count=created, updated_count=updated, finished_at=datetime.now(UTC))
        if job is None: raise LookupError("job is not running")
        return job

    def finish_failure(self, job_id: UUID, message: str) -> CollectionJob:
        job = self._transition(job_id, (JobStatus.RUNNING,), JobStatus.FAILED, error_message=message[:2000], finished_at=datetime.now(UTC))
        if job is None: raise LookupError("job is not running")
        return job

    def schedule_retry(self, job_id: UUID, message: str) -> CollectionJob:
        job = self._transition(job_id, (JobStatus.RUNNING,), JobStatus.RETRY_PENDING, error_message=message[:2000], retry_count=CollectionJobRecord.retry_count + 1)
        if job is None: raise LookupError("job is not running")
        return job

    def _transition(self, job_id, source, target, **values):
        with self.session_factory() as session:
            record = session.execute(update(CollectionJobRecord).where(CollectionJobRecord.id == job_id, CollectionJobRecord.status.in_([item.value for item in source])).values(status=target.value, **values).returning(CollectionJobRecord)).scalar_one_or_none()
            session.commit()
            return None if record is None else to_job(record)
```

`collection_tasks.py` 的工作顺序：


异常处理：

- 可重试错误：临时网络失败、上游 `5xx`、可恢复连接错误；使用指数退避。
- 不应立即重试：参数错误、专题/订阅已删除、明显的 `429` 限流；记录错误并按策略等待下个窗口。

```python
import asyncio
from uuid import UUID

from app.domain.jobs.ports import CollectionJobRepository
from app.domain.topics.models import TopicPaper
from app.domain.topics.ports import TopicRepository


class RetryableCollectionError(Exception):
    """Represent a transient upstream failure that Celery may retry."""


class RetryableCollectorError(Exception):
    """Represent a retryable failure reported by the collector adapter."""


class CollectionExecutionService:
    """执行采集、关联论文并写回任务终态。"""

    def __init__(self, jobs: CollectionJobRepository, topics: TopicRepository, papers, collector) -> None:
        self.jobs = jobs
        self.topics = topics
        self.papers = papers
        self.collector = collector

    def execute(self, job_id: UUID) -> None:
        job = self.jobs.mark_running(job_id)
        if job is None:
            return

        try:
            topic = self.topics.get(job.topic_id)
            if topic is None:
                raise LookupError(f"topic not found: {job.topic_id}")

            query = " OR ".join(topic.keywords)
            papers = asyncio.run(self.collector(query, 50))
            created, updated = self.papers.upsert_many(papers)
            for paper in papers:
                if self.topics.get_paper_link(topic.id, paper.paper_id) is None:
                    self.topics.add_paper(TopicPaper(topic_id=topic.id, paper_id=paper.paper_id))

            self.jobs.finish_success(
                job.id,
                fetched=len(papers),
                created=created,
                updated=updated,
            )
        except RetryableCollectorError as exc:
            self.jobs.schedule_retry(job.id, str(exc))
            raise RetryableCollectionError(str(exc)) from exc
        except Exception as exc:
            self.jobs.finish_failure(job.id, str(exc))
            return
```

`mark_running()` 必须是原子更新；返回空值表示任务已被另一 worker 取得或已经结束。此处复用现有 arXiv 适配器，并只在同步 Celery task 边界调用一次 `asyncio.run()`。采集器将临时网络或上游 `5xx` 转换为 `RetryableCollectorError`；服务把 Job 改为 `retry_pending`，再由 Task 调用 `self.retry()`。不可重试异常才写入 `failed`。因此重试投递回来时仍可从 `retry_pending` 原子地进入 `running`，不会被错误的终态拦截。

### 2.4 调度与增量语义

不要每次订阅运行都重新请求相同的全部论文。增量规则应明确：

1. 使用 arXiv ID 作为论文去重主键。
2. 记录订阅上次成功采集时间和最后处理的提交时间。
3. 请求结果按提交时间排序；允许少量重叠窗口，依赖去重抵抗延迟。
4. 只把新增或更新的论文关联到专题。
5. 调度器只负责创建 Job；真实采集仍由 worker 完成。

Celery Beat 的周期规则来源于 Subscription。早期可以每小时扫描一次启用订阅并创建任务，不必动态为每条订阅生成独立 cron 条目。

创建 `app/application/subscription_scheduler.py`，使 Beat 只负责扫描和创建 Job，而不直接请求 arXiv：

```python
class SubscriptionScheduler:
    """Create scheduled collection jobs for due subscriptions."""

    def __init__(self, subscriptions, jobs) -> None:
        self.subscriptions = subscriptions
        self.jobs = jobs

    def run_once(self, now) -> int:
        """Create one idempotent job per due subscription.

        :return: Number of newly created jobs.
        """
        created = 0
        for subscription in self.subscriptions.list_due(now):
            window = now.strftime("%Y%m%d%H")
            _, was_created = self.jobs.create(
                subscription.topic_id,
                idempotency_key=f"scheduled:{subscription.id}:{window}",
                trigger_type="scheduled",
            )
            created += int(was_created)
        return created
```

```python
        self.topics = topics
        self.jobs = jobs
        self.dispatcher = dispatcher
        self.documents = documents

    def create(self, topic_id: UUID, *, target_index_version: str, idempotency_key: str, document_id: UUID | None = None) -> tuple[IndexJob, bool]:
        if self.topics.get(topic_id) is None:
            raise LookupError("topic not found")
        if document_id is not None:
            if self.documents is None or self.documents.get_for_topic(topic_id, document_id) is None:
                raise LookupError("document not found in topic")

            IndexJob(topic_id=topic_id, document_id=document_id, target_index_version=target_index_version, idempotency_key=idempotency_key)

            except Exception as exc:
                self.jobs.finish_failure(job.id, f"unable to enqueue index task: {exc}")
                raise
        return job, created

    def get_or_raise(self, job_id: UUID) -> IndexJob:
        job = self.jobs.get(job_id)
        if job is None:
            raise LookupError("index job not found")
        return job
```

创建 `app/application/subscription_scheduler.py`。窗口键由订阅 ID 与 UTC 小时组成，绝不能使用永久的 `manual:{topic_id}`：

```python
from datetime import UTC, datetime


class SubscriptionScheduler:
    def __init__(self, subscriptions, collection_jobs) -> None:
        self.subscriptions = subscriptions
        self.collection_jobs = collection_jobs

    def run_once(self, now: datetime) -> int:
        utc_now = now.astimezone(UTC)
        window = utc_now.strftime("%Y%m%d%H")
        created_count = 0
        for subscription in self.subscriptions.list_due(utc_now):
            key = f"scheduled:{subscription.id}:{window}"
            _, created = self.collection_jobs.create(
                subscription.topic_id,
                idempotency_key=key,
                trigger_type="scheduled",
                subscription_id=subscription.id,
            )
            created_count += int(created)
        return created_count

    def run_once_now(self) -> int:
        return self.run_once(datetime.now(UTC))
```

```python
from datetime import datetime
from uuid import UUID

from sqlalchemy import select, update
from sqlalchemy.exc import IntegrityError

from app.domain.subscriptions.models import IndexJob, Subscription
from app.infrastructure.persistence.job_models import IndexJobRecord, SubscriptionRecord


def subscription_from_record(record: SubscriptionRecord) -> Subscription:
    return Subscription(
        id=record.id, topic_id=record.topic_id, keywords=list(record.keywords), categories=list(record.categories),
        interval_minutes=record.interval_minutes, max_results=record.max_results, enabled=record.enabled,
        last_succeeded_at=record.last_succeeded_at, last_submission_at=record.last_submission_at,
    )


class PostgresSubscriptionRepository:
    def __init__(self, session_factory) -> None:
        self.session_factory = session_factory

    def create(self, subscription: Subscription) -> Subscription:
        with self.session_factory() as session:
            record = SubscriptionRecord(**subscription.model_dump())
            session.add(record); session.commit(); session.refresh(record)
            return subscription_from_record(record)

    def get(self, subscription_id: UUID) -> Subscription | None:
        with self.session_factory() as session:
            record = session.get(SubscriptionRecord, subscription_id)
            return None if record is None else subscription_from_record(record)

    def update(self, subscription_id: UUID, **values: object) -> Subscription | None:
        with self.session_factory() as session:
            record = session.get(SubscriptionRecord, subscription_id)
            if record is None: return None
            for key, value in values.items(): setattr(record, key, value)
            session.commit(); session.refresh(record)
            return subscription_from_record(record)

    def list_by_topic(self, topic_id: UUID) -> list[Subscription]:
        with self.session_factory() as session:
            return [subscription_from_record(x) for x in session.scalars(select(SubscriptionRecord).where(SubscriptionRecord.topic_id == topic_id)).all()]

    def list_due(self, now: datetime) -> list[Subscription]:
        with self.session_factory() as session:
            records = session.scalars(select(SubscriptionRecord).where(SubscriptionRecord.enabled.is_(True))).all()
            return [x for x in map(subscription_from_record, records) if x.last_succeeded_at is None or (now - x.last_succeeded_at).total_seconds() >= x.interval_minutes * 60]

    def mark_succeeded(self, subscription_id: UUID, now: datetime) -> None:
        with self.session_factory() as session:
            session.execute(update(SubscriptionRecord).where(SubscriptionRecord.id == subscription_id).values(last_succeeded_at=now))
            session.commit()


class PostgresIndexJobRepository:
    def __init__(self, session_factory) -> None: self.session_factory = session_factory

    def create_if_absent(self, job: IndexJob) -> tuple[IndexJob, bool]:
        with self.session_factory() as session:
            existing = session.scalar(select(IndexJobRecord).where(IndexJobRecord.idempotency_key == job.idempotency_key))
            if existing is not None: return IndexJob.model_validate(existing, from_attributes=True), False
            record = IndexJobRecord(**job.model_dump())
            try:
                session.add(record); session.commit(); session.refresh(record)
            except IntegrityError:
                session.rollback()
                return IndexJob.model_validate(session.scalar(select(IndexJobRecord).where(IndexJobRecord.idempotency_key == job.idempotency_key)), from_attributes=True), False
            return IndexJob.model_validate(record, from_attributes=True), True

    def get(self, job_id: UUID) -> IndexJob | None:
        with self.session_factory() as session:
            record = session.get(IndexJobRecord, job_id)
            return None if record is None else IndexJob.model_validate(record, from_attributes=True)

    def mark_running(self, job_id: UUID) -> IndexJob | None:
        """Atomically claim pending work; a second worker receives None."""
        with self.session_factory() as session:
            result = session.execute(
                update(IndexJobRecord)
                .where(IndexJobRecord.id == job_id, IndexJobRecord.status.in_(("pending", "retry_pending")))
                .values(status="running", error_message=None)
                .returning(IndexJobRecord)
            )
            record = result.scalar_one_or_none()
            session.commit()
            return None if record is None else IndexJob.model_validate(record, from_attributes=True)

    def finish_success(self, job_id: UUID, *, indexed_count: int | None = None) -> IndexJob:
        with self.session_factory() as session:
            record = session.scalar(
                update(IndexJobRecord)
                .where(IndexJobRecord.id == job_id, IndexJobRecord.status == "running")
                .values(status="succeeded", error_message=None)
                .returning(IndexJobRecord)
            )
            if record is None: raise LookupError("index job is not running")
            session.commit()
            return IndexJob.model_validate(record, from_attributes=True)

    def finish_failure(self, job_id: UUID, message: str) -> IndexJob:
        with self.session_factory() as session:
            record = session.scalar(
                update(IndexJobRecord)
                .where(IndexJobRecord.id == job_id, IndexJobRecord.status == "running")
                .values(status="failed", error_message=message[:2000])
                .returning(IndexJobRecord)
            )
            if record is None: raise LookupError("index job is not running")
            session.commit()
            return IndexJob.model_validate(record, from_attributes=True)
```

```python
from uuid import UUID

from fastapi import APIRouter, Depends, Header, HTTPException, Response, status

from app.api.dependencies import get_collection_job_service
from app.application.collection_job_service import CollectionJobService
from app.schemas.jobs import CollectionJobResponse

router = APIRouter(tags=["collection-jobs"])


@router.post(
    "/topics/{topic_id}/collection-jobs",
    response_model=CollectionJobResponse,
)
def create_collection_job(
    topic_id: UUID,
    response: Response,
    idempotency_key: str = Header(alias="Idempotency-Key", min_length=1, max_length=160),
    service: CollectionJobService = Depends(get_collection_job_service),
) -> CollectionJobResponse:
    job, created = service.create(topic_id, idempotency_key=idempotency_key)
    response.status_code = status.HTTP_202_ACCEPTED if created else status.HTTP_200_OK
    return CollectionJobResponse.from_domain(job)


@router.get("/collection-jobs/{job_id}", response_model=CollectionJobResponse)
def get_collection_job(job_id: UUID, service: CollectionJobService = Depends(get_collection_job_service)):
    try:
        return CollectionJobResponse.from_domain(service.get_or_raise(job_id))
    except LookupError as exc:
        raise HTTPException(status_code=404, detail="collection job not found") from exc
```

### 3.1 自动化测试

### 3.3 `v0.3.0` 完成定义

- 采集、索引不在 Web 请求中执行。
- 所有 Job 状态与统计在 PostgreSQL 中可追溯。
- Redis/Celery 重启后，已完成 Job 历史仍可查询。
- 重复采集不会造成论文或专题关联重复。
- 前端能展示任务状态与失败原因。
- 新增迁移、任务和 API 测试全部通过。
- IndexJob 的状态、API 契约与阶段 3 所需的派发边界均已完成。

### 3.4 常见错误

| 问题 | 原因 | 处理 |
| --- | --- | --- |
| API 一直等待 arXiv 返回 | 在 Router 或 Service 中直接执行采集 | Router 只创建 Job，Worker 执行真实 I/O。 |
| Redis 中有结果但数据库无 Job | 把队列当作业务数据库 | PostgreSQL 是状态真相来源。 |
| 重试产生重复论文 | 缺少 arXiv ID 与 TopicPaper 唯一约束 | 用数据库约束兜底，应用层做幂等。 |
| Job 永远 `running` | Worker 崩溃未写终态 | 引入超时、心跳/过期检查和失败恢复策略。 |
| Beat 重复创建任务 | 调度扫描没有幂等窗口 | 创建 Job 时使用订阅+时间窗口的幂等键。 |

### 推荐提交边界

1. `feat: 增加订阅与任务领域模型`
