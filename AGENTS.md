# 工作区协作约定

## Git 规则

- 本仓库仅作本地 Git 管理。不得配置远程仓库、推送或执行其他对外同步操作，除非用户明确要求。
- 所有以 `.` 开头的目录均为本地状态，不得暂存或提交；提交前确认 `.gitignore` 已生效。
- 保留可复现任务所需的源码、配置和关键成果；不要提交虚拟环境、缓存、依赖目录或可重新生成的构建产物。
- 一次提交只覆盖一个任务目录，或一项明确的工作区维护事项；混合变更必须先拆分。
- 提交前依次检查 `git diff --cached --name-only`、`git diff --cached --stat` 与 `git diff --cached --name-status`。
- 提交信息必须使用 `git-commit-message-cn` 技能：采用 `<type>: <简短中文摘要>` 标题，并附 3–5 条基于暂存内容的中文说明。

## 工作方式

- 当前工作空间默认不使用 Superpowers 相关流程或 Skills，也不创建其过程文档；仅在用户明确要求时启用。
- 修改完成后报告实际改动、验证结果与尚未提交或被忽略的内容；不要把计划描述为已完成。
